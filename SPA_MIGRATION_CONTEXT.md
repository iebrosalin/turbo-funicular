# Контекст миграции на SPA (Single Page Application)

## Текущее состояние проекта

### Стек технологий
- **Бэкенд**: FastAPI (Python) + SQLAlchemy (async) + SQLite/PostgreSQL
- **Фронтенд**: HTML/Jinja2 шаблоны + Bootstrap 5 + Vanilla JavaScript (ES6 модули)
- **Архитектура**: Гибридная (серверный рендеринг шаблонов + клиентская SPA-логика)

### Проблема текущей архитектуры
Проект находится в переходном состоянии: бэкенд рендерит HTML-шаблоны через Jinja2, но фронтенд пытается работать как SPA на чистом JavaScript. Это приводит к:
- Рассинхронизации DOM и скриптов
- Ошибкам инициализации компонентов (например, `#group-tree-root not found`)
- Сложности поддержки и расширения функционала

## Функционал для переноса в SPA

### 1. Страницы (Routes)
Следующие страницы должны быть преобразованы в SPA-роуты:

| URL | Описание | Файл шаблона |
|-----|----------|--------------|
| `/` | Главная страница (index) | `index.html` |
| `/dashboard` | Дашборд со статистикой | `dashboard.html` |
| `/scans` | Управление сканированиями | `scans.html` |
| `/assets/{id}` | Детали актива | `asset_detail.html` |
| `/assets/{id}/history` | История актива | `asset_history.html` |
| `/taxonomy` | Таксономия активов | `asset_taxonomy.html` |
| `/utilities` | Утилиты (Dig, DNS) | `utilities.html` |
| `/settings` | Настройки приложения | `settings.html` |
| `/ui-kit` | UI Kit (демо компонентов) | `ui_kit.html` |

### 2. Компоненты интерфейса

#### A. Главное меню (Навигация)
- **Текущее состояние**: Горизонтальное меню в `base.html` (верхняя панель)
- **Элементы**:
  - Дашборд (`/dashboard`)
  - Сканирования (`/scans`)
  - Настройки (`/settings`)
- **SPA-требование**: Роутинг без перезагрузки страницы, подсветка активного пункта

#### B. Сайдбар (Боковая панель)
- **Текущее состояние**: Рендерится в `scans.html`, содержит два раздела
- **Разделы**:
  1. **Группы активов** (`#group-tree-root` / `#sidebar-content`)
     - Дерево групп с иерархией
     - Счетчики активов в каждой группе
     - Кнопка создания группы
     - Фильтрация по группе (параметр `?group_id=X`)
  2. **Активные задачи** (`#active-tasks-list`)
     - Список текущих сканирований
     - Статусы (Running, Queued, Pending)
     - Прогресс выполнения
- **SPA-требование**: Динамическое обновление без перезагрузки, WebSocket/SSE для статусов

#### C. Страница сканирований (`/scans`)
- **Формы запуска**:
  - Nmap сканирование (цель, порты, скрипты, аргументы)
  - Rustscan (цель, порты, запуск Nmap после)
  - Dig (DNS запросы, типы записей)
- **Таблица истории сканирований**:
  - ID, цель, тип, статус, прогресс, дата создания
  - Фильтрация и пагинация
  - Действия: просмотр, удаление, повторный запуск
- **Импорт результатов**:
  - Загрузка XML (Nmap)
  - Загрузка текстовых файлов
- **SPA-требование**: 
  - Отправка форм через AJAX/Fetch
  - Обновление таблицы без перезагрузки
  - SSE (Server-Sent Events) для обновления статусов в реальном времени

#### D. Дашборд (`/dashboard`)
- **Виджеты**:
  - Общее количество активов
  - Количество групп
  - Активные сканирования
  - Последние уязвимости
  - Графики и статистика
- **SPA-требование**: 
  - Загрузка данных через API при инициализации
  - Периодическое обновление (polling или WebSocket)

#### E. Страница актива (`/assets/{id}`)
- **Информация**:
  - Основные данные (IP, hostname, OS, роль)
  - Группы
  - Открытые порты и сервисы
  - История изменений
- **Действия**:
  - Редактирование
  - Добавление в группу
  - Запуск сканирования
- **SPA-требование**: 
  - Загрузка данных через API
  - Модальные окна для редактирования

#### F. Таксономия активов (`/taxonomy`)
- **Функционал**:
  - Управление классификацией активов
  - Создание/редактирование типов
  - Иерархия категорий
- **SPA-требование**: Drag-and-drop дерево, модальные окна

#### G. Утилиты (`/utilities`)
- **Инструменты**:
  - Dig (DNS запросы)
  - DNS массовый lookup
  - Другие сетевые утилиты
- **SPA-требование**: Формы с AJAX отправкой, отображение результатов

#### H. Настройки (`/settings`)
- **Разделы**:
  - Общие настройки приложения
  - Настройки сканеров
  - Пользователи и права (если есть)
- **SPA-требование**: Сохранение через API, валидация на клиенте

### 3. API Endpoints (Бэкенд)

#### Assets (`/api/assets`)
```
GET    /api/assets              - Список всех активов
GET    /api/assets/{id}         - Детали актива
POST   /api/assets              - Создание актива
PUT    /api/assets/{id}         - Обновление актива
DELETE /api/assets/{id}         - Удаление актива
GET    /api/assets/schema       - Схема полей для фильтрации
POST   /api/assets/count        - Подсчет по фильтру
GET    /api/assets/search       - Поиск активов
```

#### Groups (`/api/groups`)
```
GET    /api/groups              - Список всех групп
GET    /api/groups/tree         - Дерево групп с счетчиками
GET    /api/groups/{id}         - Детали группы
POST   /api/groups              - Создание группы
PUT    /api/groups/{id}         - Обновление группы
DELETE /api/groups/{id}         - Удаление группы
POST   /api/groups/{id}/assets  - Добавить активы в группу
```

#### Scans (`/api/scans`)
```
GET    /api/scans               - История сканирований
GET    /api/scans/active        - Активные сканирования
GET    /api/scans/status        - Статус очередей
GET    /api/scans/events        - SSE поток событий
POST   /api/scans/nmap          - Запуск Nmap
POST   /api/scans/rustscan      - Запуск Rustscan
POST   /api/scans/dig           - Запуск Dig
POST   /api/scans/import        - Импорт файла
POST   /api/scans/import-nmap-xml - Импорт XML Nmap
DELETE /api/scans/{id}          - Удалить сканирование
```

## План миграции на SPA

### Этап 1: Подготовка API
1. Убедиться, что все эндпоинты возвращают JSON
2. Удалить рендеринг HTML через `TemplateResponse` из бэкенда
3. Добавить CORS (уже настроено)
4. Документировать API (Swagger/OpenAPI уже есть в FastAPI)

### Этап 2: Выбор фреймворка
Рекомендуемые варианты:
- **Vue 3 + Vite + Vue Router + Pinia** (быстрый старт, простой синтаксис)
- **React + Vite + React Router + Zustand/Redux** (гибкость, экосистема)
- **Svelte + SvelteKit** (минимальный boilerplate, высокая производительность)

### Этап 3: Структура SPA приложения
```
/frontend-spa/
├── src/
│   ├── components/         # Переиспользуемые компоненты
│   │   ├── Navbar.vue

│   │   ├── ScanForm.vue

│   │   ├── Dashboard.vue

│   │   ├── AssetDetail.vue
```

### Этап 4: Реализация
1. Настроить роутинг (Vue Router / React Router)
2. Создать базовый layout (Navbar + Sidebar + Content)
3. Реализовать компонент дерева групп с API интеграцией
4. Реализовать страницу сканирований с формами и таблицей
5. Добавить SSE/WebSocket для обновления статусов в реальном времени
6. Реализовать остальные страницы по приоритету

### Этап 5: Интеграция
1. Собрать SPA приложение (`npm run build`)
2. Настроить раздачу статики через FastAPI (`StaticFiles`)
3. Настроить fallback роутинг (все запросы -> `index.html`)
4. Протестировать production сборку

## Критические требования
1. **Без перезагрузки страниц**: Все переходы через роутер
2. **Real-time обновления**: SSE или WebSocket для статусов сканирований
3. **Сохранение состояния**: При обновлении страницы сохранять фильтры, выбранную группу
4. **Адаптивность**: Мобильная версия (бургер-меню для sidebar)
5. **Обработка ошибок**: Глобальный error handler, уведомления пользователя
6. **Производительность**: Lazy loading компонентов, кэширование данных
